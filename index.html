New features

  1. Create branch epic/name from dev.
  2. Create branch dev-name/feature from epic/name.
  3. When the feature is done, create a PR to merge dev-name/feature into epic/name. Add the under development label and assign to the reviewer.
  4. The assignee should review the code, add any comments regarding: typos, suggestions, mistakes. When the comment-fix iteration is done, the assignee comments a :+1 on the PR root. This means that the PR is good to merge and it should be done by the person who created it.
  5. Continue this flow inside the epic branch, until the whole feature is complete.
  6. After that, create a PR to merge epic/name into dev, adding the ready label. This one doesn't require code review and can be merge directly.
  7. Continue this flow until all epics are done. After that, create a PR from dev to master. The name should be the release version and add the deploy label.
  8. Create a release tag with the updated version and add the changelog, separating into: features, improvements and bug fixes.

Hotfix

  1. Create branch dev-name/fix-name from master.
  2. When the fix is done, create a PR to merge dev-name/fix-name into master. Add the bug and ready labels and assign to the reviewer.
  3. The assignee should review the code, add any comments regarding: typos, suggestions, mistakes. When the comment-fix iteration is done, the assignee comments a :+1 on the PR root. This means that the PR is good to merge and it should be done by the person who created it.
  4. Create a release tag with the updated version and add the fix(es) to the changelog.
